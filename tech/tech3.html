Differences between HTML and CSS

If websites were cakes, HTML on its own would be a boring cake, and CSS would be an exciting cake.

The CSS cake couldn’t exist without the HTML cake’s basic ingredients (flour, eggs, baking powder etc). While I’m sure the HTML cake might taste great, the CSS cake looks more appetising.

It’s important to strike a nice balance between HTML structure and CSS decorations in order to have a crowd pleasing cake.

Explain control flow and loops using an example process from everyday life

Control flow is the order of executed statement. Like reading a book, we read from the top of the page to the bottom of the page so does the computer. Loops can alter control flow by repeating an instruction until a condition is reached or changed.

An example of this in daily life is eating breakfast. There is a logical order to follow.

  1. Go to kitchen
  2. Make your breakfast
  3. While breakfast is on plate - eat breakfast
  4. When there is no more breakfast on the plate - place pate in dishwasher.

The third statement loops and breaks up the control flow UNTIL the breakfast is no longer on the plate.

Describe what the DOM is and an example of how you might interact with it

The DOM is an application programming interface (API) for documents. The DOM is an object-oriented representation of the web page, which can be modified with a scripting language such as JavaScript. JavaScript, HTML and CSS are languages that the browser reads and does stuff with. But the DOM is where that stuff happens. The Dom is like a translator for programming languages and has a logical structure like a tree.

One way to think about the DOM is like if you were playing D&D.

There is already a story planned with any number of scenarios available and rules to follow depending on what information is fed to the Dungeon Master (the DOM in this instance). As a player (HTML/CSS) you create your characters and give this information to the DM so that they can create an overall “world” where many story paths are available. You are able to manipulate the outcome of the game by reacting and expressing actions that work within the game structure. This is similar to how we are able to manage and work with the DOM to access nodes that allow us to achieve what we want with our code.

Explain the difference between accessing data from arrays and objects

Functions are blocks of code that perform specific tasks. They are very useful for containing variables that are specific to the particular function, and can be used multiple times without having to write out the entire contents of the function again. You can do this by calling the function. Functions also help to organize and keep your code clean and clear.

Say you were cleaning your house and you asked your flatmate to “clean the toilet”. Your flatmate should fully know how to complete that task without you having to break it down into smaller steps. Your flatmate should have learnt how to clean toilets by now… This is what functions do. As long as we have previously explained the steps needed to do what is being asked, functions allow us to complete tasks without having to break down each component every time we want the function to run.
